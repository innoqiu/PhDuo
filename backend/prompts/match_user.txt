Inputs:

A) STUDENT PROFILE JSON (`student_profile`):
{student_profile}

B) PROFESSOR/LAB PROFILE JSON (`professor_infor`):
{professor_profile}

C) OPTIONAL: Prior professor report (`professor_report`) if available:
{professor_report}

Task:
Produce a detailed, multi-dimensional matching analysis between the student and the PI/lab, then provide an outreach/email strategy tailored to:
- student experience (education/work/projects/publications/skills)
- PI research direction + key publications/items (from professor_infor)
- PI recruitment protocol/preferences if explicitly present

EVIDENCE RULE
- Any factual claim about the PI must based on evident clue, paraphrase and presented.
- Any factual claim about the student must based on evident clue, paraphrase and presented.
- Any claim derived from professor_report must be explained in logic.
- For every inference: include Confidence (0–100), Rationale (1–2 sentences), Evidence paths used.
- If something cannot be concluded: mark Unknown and specify what evidence is missing.

Output:
Return a single Markdown report with these exact section headers and order:

## 0. Match Snapshot (≤12 bullets)
- Overall fit rating (0–5) + Confidence
- Top 3 match strengths (evidence pointers)
- Top 3 gaps/risks (evidence pointers)
- 2 “fast verification” items (what to confirm externally)

## 1. Data Quality & Constraints
1.1 Student data completeness (missing fields and impact)
1.2 Professor data completeness + freshness (use `stability_signals.freshness_indicators`)
1.3 Hard constraints detected (e.g., location, degree timing, required materials, deadlines) with evidence paths

## 2. Multi-Dimensional Fit Analysis (score each 0–5)
For each dimension below, output in the same format:
- Score (0–5)
- Evidence (paths + short paraphrase)
- Inference (Confidence + Rationale + Evidence paths)
- Risks/Unknowns
- Recommended actions (for student)

### 2.1 Topic & Problem Alignment
Use `research_signals.research_interests`, `application_or_problem_domains`, and `research_items`.

### 2.2 Methods/Research Style Alignment
Compare PI style (infer from research_items/venues/descriptions) with student experience (projects/skills/publications).

### 2.3 Skills Readiness & Research Maturity
Match PI needs implied by work with student’s demonstrated skills and outputs.

### 2.4 Mentorship Style Compatibility (hands-on ↔ hands-off)
Infer PI guidance style cautiously from available signals (team composition, protocols, publication patterns if present) and compare to student needs.

### 2.5 Project Breadth Preference Compatibility
Portfolio-style vs deep-focus: infer from PI topics breadth and compare with student preferences/trajectory.

### 2.6 Recruitment & Funding Feasibility
Only use `recruitment_signals.* knowables` and `stability_signals.funding_sources` + activity freshness. If absent: Unknown.

### 2.7 Lab Culture & Working Norms Fit (cautious)
Use `team_raw.*`, alumni outcomes, mentoring signals, and explicit protocol cues.

### 2.8 Career Pipeline Alignment
Compare alumni/current placements (if present) with student goals.

### 2.9 Communication & Responsiveness Risk (cautious)
Use freshness indicators + any “ghosting” signals if present; otherwise Unknown.

## 3. Gap-to-Plan (Actionable)
3.1 Top 5 gaps to close (ranked) with evidence
3.2 How to position the student’s background to this PI (3–6 bullets)
3.3 Best “2 paper anchors”: pick 1–2 PI research_items that are the strongest bridge to the student, and explain why (evidence + inference)

## 4. Outreach & Email Strategy (Tailored)
Important: If PI-specific instructions exist in `recruitment_signals.application_protocol` or `required_materials`, follow them and highlight them first.

4.1 PI-specific outreach preferences (ONLY if explicit; otherwise say “No explicit preference found”)
4.2 Best practices for this case (concise, specific, clear ask) tailored to the student/PI
4.3 What to mention (must include: mutual interests + 1–2 concrete PI items + 1–2 concrete student proofs)
4.4 What to avoid (generic praise, long life story, attaching huge files, demanding supervision)
4.5 Attachments checklist (CV, transcript, writing sample, etc.) based on required_materials

Return only the report.
